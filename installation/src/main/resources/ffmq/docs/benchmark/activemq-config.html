ActiveMQ 5.4.2

Installation steps used for this benchmark :

- Unzipped the default install
- Disabled flow control and enabled VM cursors for destinations :
        <destinationPolicy>
            <policyMap>
              <policyEntries>
                <policyEntry topic=">" producerFlowControl="false" memoryLimit="50mb">
                  <pendingSubscriberPolicy>
                    <vmCursor />
                  </pendingSubscriberPolicy>
                </policyEntry>
                <policyEntry queue=">" producerFlowControl="false" memoryLimit="50mb">
                  <pendingQueuePolicy>
                    <vmQueueCursor/>
                  </pendingQueuePolicy>
                </policyEntry>
              </policyEntries>
            </policyMap>
        </destinationPolicy>
- Increased memory limits 
        <systemUsage>
            <systemUsage>
                <memoryUsage>
                    <memoryUsage limit="300 mb"/>
                </memoryUsage>
                <storeUsage>
                    <storeUsage limit="1 gb"/>
                </storeUsage>
                <tempUsage>
                    <tempUsage limit="100 mb"/>
                </tempUsage>
            </systemUsage>
        </systemUsage>
- Used the amqPersistenceAdapter when testing non-persistent messaging !
(Leaving kahaDB enabled was unexpectedly slowing down non-persistent messaging, perhaps because messages are indexed to disk even if non-persistent ? weird...)
